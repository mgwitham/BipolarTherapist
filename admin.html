Published Listings

Purpose
Control how live listings are staged, promoted, and maintained once they are already public.
Use This When
You are deciding which listings should be launch-ready, featured, refreshed, or left alone.
Done Means
The listing has the right launch state and is not missing obvious trust or freshness work.
Operator Playbook
Good First Move
Start with listings that are already close to launch-ready or are carrying a visible freshness risk. Avoid tinkering with stable listings that do not need a decision today.
Strong Decision Example
If a listing has strong profile depth, current trust signals, and no obvious blocker, promote it. If the profile is aging or missing important operational detail, send it into refresh or confirmation instead of featuring it.
Avoid
Do not feature or elevate a listing just because the profile sounds good. The operator standard is live trust, not intuition.

Refresh Queue

Source-backed profiles that are starting to age or still need operational re-confirmation.

Purpose
Work live listings whose operational truth is aging or whose trust fields need attention.
Use This When
A listing is already public but should not stay untouched because freshness or trust quality is slipping.
Done Means
The listing has been refreshed, confirmed, or consciously deferred with a clear reason.
Operator Playbook
Good First Move
Open the oldest or riskiest listing first, confirm what is stale, and decide whether you can refresh from a source, need therapist confirmation, or should defer with a reason.
Strong Decision Example
If a website clearly shows updated insurance, telehealth, or availability, refresh the profile directly. If the source is unclear for a trust-critical field, move it into confirmation instead of guessing.
Avoid
Do not leave a listing sitting in refresh just because you cannot finish it in one pass. Either update it, defer it, or push it to confirmation.

Import Blocker Sprint

The top strong-warning profiles still blocking the strict safe-import gate.

Purpose
Clear the highest-value missing fields that prevent safe import and trustworthy live data.
Use This When
A profile is blocked on trust-critical data like license, insurance, wait time, or bipolar experience.
Done Means
The blocker is cleared, moved into confirmation follow-up, or intentionally left with the next move recorded.
Operator Playbook
Good First Move
Pick the profile with the fewest unresolved blockers and clear the missing trust-critical fields one by one. This lane is about unlocking safe imports, not polishing every detail.
Strong Decision Example
If you can verify license and operating details from a strong source, clear the blocker. If the missing field requires therapist input, move it to confirmation with a note on exactly what still needs an answer.
Avoid
Do not leave a blocker unresolved without a next step. Every blocked profile should end in cleared, escalated to confirmation, or deliberately deferred.

Confirmation Sprint

The top therapist-confirmation wave to work next, condensed into a smaller send-ready packet.

Purpose
Work the next small batch of therapist confirmation outreach without scanning the entire queue.
Use This When
You want a focused outreach packet to send, follow up on, or advance toward confirmed status.
Done Means
Each sprint profile has moved from not started to sent, waiting, confirmed, or applied.
Operator Playbook
Good First Move
Treat this as a small outbound packet. Work the batch top to bottom, update the state on each profile, and stop once every row has a clear next status.
Strong Decision Example
If you have enough information to send outreach, send it and mark the profile as sent. If a reply already came back, capture the answers and advance it to confirmed or applied instead of re-sending.
Avoid
Do not use the sprint like a passive reading list. Every profile in the batch should leave the session in a different and more useful state.

Confirmation Queue

Live profiles that still need therapist-confirmed answers for high-value unknowns.

Purpose
Manage the full lifecycle of confirmation work across all live listings.
Use This When
You need to send requests, track reply state, save confirmed values, or mark live updates as applied.
Done Means
The profile is either waiting on therapist reply, fully confirmed, or already applied back into live data.
Operator Playbook
Good First Move
Filter by status, then work one state at a time. Not-started rows need outreach, waiting rows need follow-up judgment, and confirmed rows need values applied back into the live profile.
Strong Decision Example
If a therapist replied with specific answers, save the confirmed values and mark what was applied. If there is no response yet, keep the profile in a waiting state rather than pretending the data is settled.
Avoid
Do not mark confirmation work as done just because outreach was sent. The work is only complete when the status accurately reflects what happened next.

Applications

Applications from the public signup flow appear here when the review API is running.

Purpose
Review therapist-submitted applications and decide whether they should be approved, revised, rejected, or published.
Use This When
A therapist has submitted or updated a profile and it still needs a human review decision.
Done Means
The application leaves pending limbo with a clear state: approved, needs changes, rejected, or published.
Operator Playbook
Good First Move
Review the application for trust, completeness, and publishability in that order. The job is to make a clean decision, not to keep applications sitting in indefinite review.
Strong Decision Example
If the therapist clearly meets quality and trust standards, approve or publish the application. If key details are weak but fixable, request changes with a concrete reason instead of rejecting it vaguely.
Avoid
Do not hold a decent application in reviewing status just because it is not perfect. Choose the next state and move it forward.

Candidate Review Queue

Sourced therapist candidates land here before they become live listings. Review duplicates, trust readiness, and publish decisions in one place.

Purpose
Turn sourced therapist leads into publishable listings or reject them cleanly before they create clutter.
Use This When
You are adding supply, checking duplicates, or deciding whether a sourced therapist is ready for live review.
Done Means
The candidate is published, moved to confirmation, merged as a duplicate, or archived with a clear reason.
Operator Playbook
Good First Move
Check for duplicates first, then judge whether the candidate is strong enough to publish now, needs confirmation, or should be archived. This keeps low- quality supply from cluttering the system.
Strong Decision Example
If the candidate looks unique and has enough trustworthy detail, move it toward publish-ready. If it is promising but missing a crucial operational fact, send it to confirmation instead of forcing a publish decision.
Avoid
Do not keep marginal candidates around without a reason. Every candidate should become a listing, become a follow-up item, merge, or leave the queue.
