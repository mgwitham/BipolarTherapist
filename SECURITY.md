# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security problems.

Instead, report vulnerabilities privately to the project owner with:

- a short summary of the issue
- steps to reproduce it
- the impact you believe it has
- any suggested fix or mitigation

If the issue involves exposed credentials, revoke or rotate them immediately before reporting.

## Response Expectations

We will aim to:

- acknowledge the report promptly
- reproduce and assess the issue
- ship a fix or mitigation as quickly as practical
- disclose the issue publicly only after it is fixed or understood
